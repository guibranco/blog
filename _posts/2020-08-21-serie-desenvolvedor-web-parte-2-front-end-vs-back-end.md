<div class="section-header">
  <div class="section-num">01</div>
  <div class="section-title-wrap"><h2>Front-end</h2></div>
</div>

O front-end é a parte visível ao usuário — a interface gráfica com a qual ele interage. No contexto web, é composta por três tecnologias obrigatórias:

<div class="providers-grid">
  <div class="provider-card"><div class="provider-name">HTML</div><div class="provider-detail">Estrutura o documento e o conteúdo. Não é linguagem de programação, mas é indispensável.</div></div>
  <div class="provider-card"><div class="provider-name">CSS</div><div class="provider-detail">Define a apresentação visual — responsividade, animações, temas.</div></div>
  <div class="provider-card"><div class="provider-name">JavaScript</div><div class="provider-detail">Linguagem de programação do navegador. Manipula o DOM e cria interatividade.</div></div>
</div>

**Bibliotecas JS populares (2020):** React, Angular, Vue, jQuery (legado).

**Frameworks CSS:** Bootstrap, Materialize, Bulma, Foundation, Semantic.

Além do domínio técnico, conhecimentos extras que fazem diferença: UI/UX, SEO, paleta de cores, testes A/B, Sass/Less, BEM, SPA e PWA.

<div class="divider">· · ·</div>

<div class="section-header">
  <div class="section-num">02</div>
  <div class="section-title-wrap"><h2>Back-end</h2></div>
</div>

O back-end é a parte invisível — onde acontece o processamento, conexões com banco de dados, envio de e-mails e as regras de negócio. **Tudo que está no front-end pode ser alterado pelo usuário**, por isso validações críticas devem sempre existir no back-end também.

<div class="callout callout-warn">
  <div class="callout-label">Mito a combater</div>
  Não acredite em vídeos ou artigos falando que linguagem X ou Y "vai morrer". Isso são pessoas direcionando você para uma linguagem que preferem, ou pura ignorância do mercado.
</div>

<div class="providers-grid">
  <div class="provider-card"><div class="provider-name">PHP</div><div class="provider-detail">Baixa curva de aprendizado. Muito usado em empresas menores, agências e e-commerces. Grande volume de vagas para júnior.</div></div>
  <div class="provider-card"><div class="provider-name">JavaScript (Node.js)</div><div class="provider-detail">O mesmo JS do front-end, agora no back. Funciona também em desktop (Electron) e mobile. Curva de aprendizado baixa.</div></div>
  <div class="provider-card"><div class="provider-name">Python</div><div class="provider-detail">A mais fácil de aprender, mas poucas vagas web para júnior. Forte em dados e ML.</div></div>
  <div class="provider-card"><div class="provider-name">C# (.NET)</div><div class="provider-detail">Usado em empresas médias/grandes, fintechs e sistemas corporativos.</div></div>
  <div class="provider-card"><div class="provider-name">Java</div><div class="provider-detail">Dominante em bancos, indústria e grandes empresas.</div></div>
  <div class="provider-card"><div class="provider-name">Rust / Go / Elixir</div><div class="provider-detail">Foco em performance. Menos vagas para iniciantes, mas crescimento acelerado.</div></div>
</div>

<div class="callout callout-tip">
  <div class="callout-label">Minha recomendação pessoal</div>
  Comecei em PHP em 2007. Se fosse começar hoje, escolheria <strong>JavaScript (Node.js)</strong> — funciona no front, no back, no desktop e no mobile. Uma linguagem, múltiplos mundos.
</div>

<div class="section-header">
  <div class="section-num">04</div>
  <div class="section-title-wrap"><h2>O que estudar para começar</h2></div>
</div>

- **Front-end:** HTML → CSS → JavaScript → uma biblioteca (React ou Vue) → um framework CSS (Bootstrap). Stack dominada em 3 meses a 1 ano de estudos consistentes.
- **Back-end:** PHP ou JavaScript (Node.js) para entrar mais rápido. C# ou Java para empresas maiores, com curva de aprendizado maior.
- **Em qualquer caminho:** de 6 meses a 2 anos para estar apto ao primeiro emprego.

<div class="conclusion">
  <h2>A barreira do primeiro emprego</h2>
  <p>O mercado é competitivo para quem está entrando — isso é real. Mas depois que você quebra a barreira do primeiro emprego, fica muito mais fácil. O que importa nessa fase não é o salário, é o aprendizado.</p>
  <p>Escolha a área que mais te atrai, estude com consistência, construa um portfólio com projetos reais e fuja de atalhos. Não existe caminho mágico.</p>
